# Voyager Manual Test Checklist

## 目的

本清单用于验证真实 Chrome 扩展行为。自动语法检查和源码字符串检查不能替代本清单。

每个用例必须记录：

- 状态：`PASS`、`FAIL`、`BLOCKED` 或 `NOT RUN`
- Chrome 版本
- 扩展加载来源：源码目录或 `dist/`
- 测试对话 URL 或可识别说明
- 对话规模
- 截图、录屏、控制台错误或补充说明

## 测试前准备

1. 运行：

   ```powershell
   npm run check
   git status --short --branch
   ```

2. 打开 `chrome://extensions`，开启开发者模式。
3. 优先选择仓库源码目录加载当前工作区。
4. 如果测试 `dist/`，必须先明确允许重建该忽略目录，再运行 `npm run build`。
5. 打开扩展详情页并确认权限只包含当前 Manifest 声明的权限。
6. 准备以下测试数据：
   - 短对话：至少 3 个用户问题。
   - 长对话：建议至少 30 个用户问题，并能触发历史消息加载或虚拟化。
   - 新建未保存对话。
   - 包含代码块、行内公式和块级公式的对话。

## P0 核心验收

### M-01 扩展加载与基础冒烟

步骤：

1. 加载扩展。
2. 打开或刷新一个 `https://chatgpt.com/*` 对话。
3. 检查页面和开发者工具控制台。

预期结果：

- 页面正常加载，ChatGPT 输入和滚动不被阻塞。
- 有用户问题时保留 ChatGPT 官方右侧横线导航，并出现 Voyager 菜单入口。
- Voyager 不额外绘制第二套右侧问题横线。
- 没有重复 Voyager UI。
- 控制台没有持续重复的未捕获错误。

### M-02 用户问题识别与去重

步骤：

1. 打开短对话。
2. 对照页面中实际用户问题数量，检查目录条目。
3. 检查助手回复、输入框、按钮和 Voyager UI 是否被误识别。

预期结果：

- 每个用户问题只出现一次。
- 助手回复和非问题 UI 不出现在问题目录中。
- 目录问题顺序与对话顺序一致。

### M-03 官方右侧横线与 Voyager 菜单共存

步骤：

1. 分别点击 ChatGPT 官方右侧横线导航的首个、中间和末尾问题。
2. 点击或悬停官方导航中部附近的 Voyager 菜单入口。
3. 对照官方横线跳转落点和 Voyager 菜单打开状态。

预期结果：

- 官方横线每次跳转到对应用户问题，而不是相邻助手回答。
- Voyager 不显示第二套横线 marker。
- Voyager 菜单入口可打开插件目录和导出功能。

### M-04 目录点击定位

步骤：

1. 打开目录。
2. 分别点击首个、中间和末尾问题。

预期结果：

- 每次跳转到对应用户问题。
- 点击期间有明确状态，完成后目录按当前交互设计关闭。
- 不出现跳转到其他会话缓存的问题。

### M-05 官方横线导航滚轮与插件非干扰

步骤：

1. 打开长对话。
2. 将鼠标放在 ChatGPT 官方右侧横线导航上，从首个问题持续向下滚到末尾，再滚回首个。
3. 观察是否出现 Voyager 自绘横线或插件拦截官方滚轮。

预期结果：

- 只有 ChatGPT 官方横线可见，Voyager 不再显示自绘横线。
- 插件菜单入口不阻塞官方横线滚动和点击。
- 页面不出现两套横线叠加。

历史关联：

- `2026-05-27 - 滑轨滚轮向下滚动回弹`

### M-06 目录完整性与状态

步骤：

1. 在长对话的顶部、中部和底部分别打开目录。
2. 比较已记录问题数量和当前 DOM 已加载问题数量。
3. 检查暂时无法定位的问题状态。

预期结果：

- 目录保留所有已记录问题，而不只显示当前 DOM 可见问题。
- 当前不可可靠定位的问题明确显示“需重新采集”等状态。
- Voyager 目录展示所有已记录问题；官方右侧横线由 ChatGPT 自身负责展示和跳转。

历史关联：

- `2026-05-29 - 三横线目录只显示部分问题`

### M-07 目录搜索与一句话概览

步骤：

1. 搜索一个只出现在问题文本中的关键词。
2. 搜索一个只出现在一句话概览中的关键词。
3. 检查有回复、无回复和历史未加载问题。

预期结果：

- 搜索能匹配问题文本和当前可用一句话概览。
- 每条辅助信息标记为“一句话概览”，页面没有独立“总结”标签。
- 未加载或未识别到回答时显示明确占位说明，不伪装为已总结内容。

历史关联：

- `2026-06-15 - 删除无效且冗余的总结提示词流程`
- `2026-06-15 - 回答预览只截取开头，无法覆盖前后内容`

### M-08 实时采集

步骤：

1. 在当前对话新增一个用户问题并等待助手回复。
2. 不刷新页面，观察官方右侧横线、Voyager 菜单入口和目录。
3. 暂停实时采集，再新增问题；随后重新开启实时采集。

预期结果：

- 开启实时采集时，新问题自动进入索引。
- 暂停时状态明确，重新开启后可以补充扫描。
- 不出现重复问题。

### M-09 深度采集与取消

步骤：

1. 在长对话中记录当前滚动位置。
2. 启动深度采集并观察状态。
3. 至少执行一次取消，再重新执行一次完整采集。

预期结果：

- 采集状态和问题数量持续更新。
- 取消能够停止流程。
- 完成或取消后恢复到原滚动位置附近。
- 页面没有持续自动滚动或卡死。

### M-10 缓存恢复与会话隔离

步骤：

1. 在保存的长对话中完成部分或完整采集。
2. 刷新页面，检查问题索引恢复。
3. 切换到另一个对话，再切回原对话。

预期结果：

- 原对话的问题索引可以恢复。
- 不同对话的问题不会互相污染。
- 缓存条目状态与是否实际加载一致。

### M-11 新建会话转持久会话

步骤：

1. 从 `https://chatgpt.com/` 新建未保存对话。
2. 发送多个问题，等待 URL 转为 `/c/<id>`。
3. 刷新并检查问题索引。

预期结果：

- 未保存会话的问题可以迁移到持久会话 ID。
- 不创建明显重复问题。
- 不污染其他新建或已保存会话。

### M-12 Markdown 复制与下载

步骤：

1. 打开导出标签。
2. 测试全选、清空和选择部分消息。
3. 执行复制 Markdown 和导出 Markdown。

预期结果：

- 只包含当前选择的已加载消息。
- 用户和助手角色顺序正确。
- 下载文件可打开，来源 URL 和导出时间存在。
- 未加载历史消息不会被描述为已经导出。

## 增强能力验收

这些用例不阻塞 MVP，但必须在声称相应能力稳定前执行。

### M-20 PDF 导出

- 选择包含普通文本、代码块和公式的消息。
- 导出 PDF。
- 预期：打印预览可打开，角色、文本、代码块和公式保持可读。

### M-21 PNG 导出

- 分别导出短内容和较长内容。
- 预期：图片可下载，内容不重复、不缺段；过高内容显示明确限制提示。

### M-22 公式复制

- 测试 KaTeX 行内公式、块级公式和选中文本。
- 分别复制 Word UnicodeMath、WPS LaTeX、Markdown LaTeX 和原始 LaTeX。
- 预期：来源可靠时格式正确；来源不可靠时有明确提示。
- 在实际 Word 和 WPS 中粘贴验证，不能只检查剪贴板文本。

### M-23 Popup 文件夹

- 新建和删除文件夹，将当前 ChatGPT 会话保存到文件夹。
- 预期：Popup 重开后数据仍存在，当前会话显示已保存。

### M-24 Options 提示词库

- 新增和删除提示词，刷新 Options 页面。
- 预期：数据保存在本地。
- 注意：当前未验证提示词能够在 ChatGPT 页面直接使用。

## 已知半连接能力确认

以下项目当前应记录为产品差距，不应在测试中误判为已完成闭环：

- `panelOpen`、`autoRefresh`、`showTimeline`、`showPromptDock`、`exportFormat` 和 `panelPosition` 是否真正影响内容脚本。
- Popup 中 JSON 导出选项是否有对应导出实现。
- 文件夹是否支持浏览和打开已保存会话。
- 提示词库是否支持页面内一键插入。
- `chat.openai.com` 是否受支持；当前 Manifest 只匹配 `chatgpt.com`。

## P0 退出条件

- M-01 至 M-12 全部为 `PASS`。
- 所有 `FAIL` 都已进入 `REGRESSION_LOG.md` 或明确列为阻塞项。
- 重新运行 `npm run check` 和 `git diff --check` 后仍通过。
- `docs/GOAL_STATUS.md` 已更新为实际测试结果和证据。
